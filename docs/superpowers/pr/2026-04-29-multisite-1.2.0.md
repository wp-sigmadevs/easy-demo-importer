## Summary

Adds full WordPress Multisite support to Easy Demo Importer. Plugin works in single-site (unchanged), per-subsite activated, or network-activated mode. Each subsite imports independently with per-blog tables. Super Admins get a dedicated Network Admin screen for cross-network status, JSON-editable demo config override, and one-click network-wide plugin installation. Subsite admins lacking install permissions are guided to ask the Network Admin.

## Why

Existing plugin assumed single-site: activation created tables only on main blog, uninstall leaked per-subsite data, and required-plugins flow had no path for subsite admins on locked-down multisite networks. Agencies running multisite couldn't deploy this plugin cleanly. v1.2.0 closes those gaps.

## Architecture

- **Hybrid activation** ŌĆö no `Network: true` header, supports both per-site and network-active.
- **`ContextResolver`** ŌĆö single source of truth for `is_multisite`/cap checks/blog targeting.
- **`NetworkInstaller`** ŌĆö `switch_to_blog` + `dbDelta` wrapper, chunked enumeration of `get_sites()`.
- **Per-blog tables** ŌĆö `wp_{N}_sd_edi_taxonomy_import` created lazily on first import, on `wp_initialize_site`, and chunked-bulk on activation.
- **Network override** ŌĆö `sd_edi_network_config` site_option resolves before per-subsite theme filter when `sd_edi_network_override_enabled` is true.
- **Network Admin UI** ŌĆö under Network Themes, tabbed React app (Dashboard / Network Config / Settings).
- **Tiered install** ŌĆö subsite admin missing plugins ŌåÆ "Notify Network Admin" mailto + Refresh; Super Admin ŌåÆ "Install all on network" button.
- **Domain-echo reset confirm** ŌĆö multisite + reset = controlled modal requiring exact subsite host echo.
